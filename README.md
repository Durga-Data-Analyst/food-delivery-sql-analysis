# Food Delivery Data Analysis using SQL

## 📌 Project Objective
Analyze food delivery transactional data to uncover revenue trends, customer behavior patterns, and operational insights.

---

## 🛠 Tools Used
- PostgreSQL
- SQL (Aggregations)
- Data Modeling

---

## 📊 Key Business Questions Answered

1. What is the total revenue generated?
2. Who are the top 5 customers by spending?
3. What are the peak order hours?
4. Which restaurants generate highest revenue?
5. What is the average delivery time?

---

## 🔍 Key Insights

- Majority of orders occur between 7 PM – 9 PM
- Top 10% customers contribute majority revenue
- Weekend orders are significantly higher
  
-  ## 🗄 Database Design

The project uses a relational schema with:
- Customers
- Restaurants
- Orders

Relationships:
- Each order is linked to one customer
- Each order is linked to one restaurant

 ## 📂 Dataset
The dataset contains transactional order data including:
- Customer ID
- Restaurant ID
- Order Amount
- Order Time
- Delivery Time
  
 ## 🚀 Outcome
This analysis helps businesses:
- Optimize delivery operations
- Target high-value customers
- Improve peak-hour staffing
